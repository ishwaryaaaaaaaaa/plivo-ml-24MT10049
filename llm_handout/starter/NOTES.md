# NOTES

**Final configuration:** BPE tokenizer (vocab 2048, trained only on `train_corpus.txt`) + RoPE positional encoding + tied token/output embeddings + GPT-2-style depth-scaled init + AdamW (weight decay 0.1, grad clip 1.0) + linear warmup (100 steps) → cosine decay (peak lr 3e-3) + block_size 320 + batch 32, trained for the full 2,000-step budget.

**Why it works:** the byte-level baseline wasted context on Devanagari, which costs 2–3 raw bytes per character, so BPE alone bought back a large chunk of effective sequence length on the Hindi half of the corpus (Run 1→3 in RUNLOG.md: 2.9294→2.3782 dev bpb). The training-recipe fix (schedule, decay, clipping, tying, init) was an independent, similarly-sized win (Run 1→2: 2.9294→2.4059), because the baseline's constant, undecayed LR was leaving the model bouncing around late in training. RoPE was the single largest lever found (Run 4→5: 2.2373→1.8770) and is free under the parameter cap, since it has zero learned weights, unlike a learned position-embedding table that has to learn every row from scratch inside a 2,000-step budget. Finally, since only optimizer *steps* are capped, not wall-clock or tokens, increasing block_size and batch size let the model see more real data per step, and the two effects stacked close to additively (Run 12: 1.8124 dev bpb, best of the sweep). Learning rate was swept and 3e-3 confirmed near-optimal in both directions.
