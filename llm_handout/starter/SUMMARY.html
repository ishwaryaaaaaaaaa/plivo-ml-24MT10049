Architecture

A small decoder-only Transformer: byte/BPE token embedding, rotary positional encoding, four pre-norm transformer blocks (causal self-attention + GELU MLP with residual connections), final LayerNorm, and a tied output head.

What Changed vs. Baseline

🔤 Tokenizer

Byte-level (vocab 256) → BPE (vocab 2048) trained only on train_corpus.txt. Directly targets Devanagari's 2–3-byte-per-character inflation.

📐 Positional Encoding

Learned absolute embeddings → RoPE. Zero learned parameters, biggest single lever found.

🔗 Weight Tying

Off → on. Reclaims ~41K parameters from a redundant output projection.

🎛️ Optimizer & Schedule

Constant-LR Adam → AdamW with decoupled weight decay, linear warmup, cosine decay, gradient clipping.

🧮 Init

Flat N(0, 0.05) → GPT-2-style init with residual-stream projections scaled by 1/√(2·n_layer).

📦 Block Size & Batch

128 / 8 → 320 / 32 — more real tokens per optimizer step, since only step count is capped.

The Ambitious Swing

Tried something risky, watched it fail, fixed it

The first byte-level BPE encoder rescanned the entire token sequence per merge — O(n × merges). It worked on short strings and effectively hung on the full ~7.3MB corpus during a round-trip test. Diagnosis: the algorithm's complexity, not its logic, was the problem. Fix: rewritten using a doubly-linked-list + min-heap of candidate merges (O(n log n)), which encodes the full corpus in about a minute and was then verified lossless on the entire corpus, the dev set, and adversarial UTF-8 input (emoji, mixed scripts).

Claude Code was used throughout, per the assignment's explicit allowance for AI coding assistants. The choice of which levers to pull (tokenizer, positional encoding, recipe, block/batch sizing) and the decision to isolate each one with a dedicated ablation run was human-directed; the resulting implementation, the ablation harness (ablate.sh/ablate2.sh), and the fix for the BPE encoder's complexity bug were AI-assisted, with the diagnosis and validation reviewed and confirmed against RUNLOG.md at each step.

Why This Is The Best Configuration Found

Three independent levers — tokenizer, training recipe, and positional encoding — were tested in isolation before being combined, and each produced a real, non-overlapping improvement: the recipe fix alone cut dev bpb by 18% relative, the tokenizer swap alone by 19% relative, and RoPE on top of both by a further 16% relative — while RoPE simultaneously reduced parameter count versus a learned position table. Block size and batch size were then swept independently and found to combine close to additively, confirming they capture distinct sources of improvement (more real context per prediction vs. less noisy gradients). Learning rate was swept last and confirmed the existing 3e-3 default was already near-optimal in both directions.
